## AI Synthesis Components in the Pipeline — Complete Catalog

### 1. `scripts/old/synthesize_corpus.py` — AI Corpus Definition Generator (Tier 4)

**What it does:**
Generates scholarly corpus definitions (shortGloss + fullDefinition) for all ~5,624 NT Greek lexemes using Claude Haiku Batch API. For each lexeme, it assembles a prompt containing NT occurrences in context, up to 6 lexicon definitions (UGL, Abbott-Smith, Thayer, Mounce, LSJ, Wiktionary), and AGDT classical attestations. The AI applies a "Three-Point Anchor" methodology (Hebrew root -> LXX bridge -> NT arrival).

**Output:**
- Writes to `data/corpus-lexicon.json` (JSON file per Strong's ID: `{shortGloss, fullDefinition}`)

**DB table written to:**
- Indirectly via admin endpoint `POST /admin/import-corpus?reset=true` which writes to `lexeme_meaning` with `source='corpus'`
- The corpus import endpoint also rebuilds `lexeme_sense` and removes stale `token_sense_override` rows

**AI-based:**
- **Model:** `claude-haiku-4-5-20251001` (Anthropic Batch API)
- **Provider:** Anthropic

**Build-time or read-time:** Build-time (offline batch)

**Status:** **Legacy** — The script is in `scripts/old/`. CLAUDE.md explicitly says "Never use Anthropic Batch API" and "Use Groq or another free service." The output file `corpus-LITE.json` is still the live source, but re-running this script is deprecated.

### 2. `scripts/old/generate_lite_glosses.py` — LITE Gloss Generator (B1 + B2)

**What it does:**
A two-pass system that produces the "LITE" (morphologically-aware) English gloss for every NT token (~137K tokens).

**Pass 1 (B1 — rule-based):**
- For each token, reads the resolved sense gloss from `lexeme_sense` (or `token_sense_override` for polysemous tokens) and applies English morphology transformation rules to produce the correct inflected form.
- Handles: verb inflection (tense/mood/voice/person/number), noun inflection, function word overrides, article glossing, personal pronoun case inflection, G846 auto;s attributive/intensive/pronoun disambiguation, syntactic context checks for ei)mi/gi/nomai.

**Pass 2 (B2 — AI refinement):**
- Finds verb combinations where B1 produced potentially awkward results (aorist, perfect, pluperfect tenses not already handled by `IRREGULAR_VERBS`)
- Submits them to Claude Batch API for correction
- Updates `gloss_set_entry` rows with AI-corrected glosses

**Output:**
- Produces per-token glosses stored in `gloss_set_entry` (gloss_set name = `lite_auto`)

**DB table written to:**
- **`gloss_set_entry`** — direct writes via `upsert_glosses()`
- **`gloss_set`** — reads `gloss_set` to find `lite_auto` set ID

**AI-based:**
- **B2 only:** Uses `claude-haiku-4-5-20251001` (Anthropic Batch API)
- B1 is fully rule-based (no AI)

**Build-time or read-time:** Build-time (offline)

**Status:** **Legacy** — The script is in `scripts/old/`. The Java-based `InflectionEngineService` (Handler D port) now runs this at build-time via `POST /admin/regenerate-lite-glosses`. The B2 refinement path using Anthropic is deprecated. However, the `derive_gloss()` logic from this script is still the basis for the Java port.

### 3. `ContextualGlossService.java` — Layer 2.5 AI Clause-Aware Gloss

For each verse, it sends all content words (non-function words) to an AI with their short_gloss definitions and morphology keys, asking the AI to produce the best English rendering in context. Uses a two-step staging workflow: (1) Stage to `ai_staging/` JSON file, (2) Commit to DB after review.

**Is it AI-based?** **Yes.**

**DB table written to:**

**Build-time or read-time:** Build-time (admin-triggered via `POST /admin/contextual-gloss/chapter` or `POST /admin/contextual-gloss/all`)

### 6. The `gloss_set_entry` Table

**Definition (V16):**
```sql
CREATE TABLE gloss_set_entry (
    id             SERIAL PRIMARY KEY,
    gloss_set_id   INTEGER NOT NULL REFERENCES gloss_set(id) ON DELETE CASCADE,
    verse_word_id  INTEGER NOT NULL REFERENCES verse_word(id) ON DELETE CASCADE,
    gloss          TEXT NOT NULL,
    UNIQUE (gloss_set_id, verse_word_id)
);
```
**Later additions (V19, V21):** Added columns `sense_index SMALLINT`, `method VARCHAR(30)`, `confidence VARCHAR(10)`.

**What it is:** A per-token, precomputed cache of glosses belonging to a named `gloss_set`. The primary set is `lite_auto` (seeded in V16). For non-English languages, sets are named `lite_{lang}` (e.g., `lite_es`).

**How it is populated:**
1. **`generate_lite_glosses.py` (B1)** — Writes directly for all tokens with rule-based English glosses
2. **`run_c1.py`** — Updates `sense_index` and gloss for polysemous tokens
3. **`generate_lite_glosses.py --b2-collect`** — Updates glosses after AI correction
4. **`LanguageGlossService.expandToCache()`** — Populates `lite_{lang}` sets via SQL expansion from `cluster_gloss_rule`
5. **`InflectionEngineService.regenerateLiteGlosses()`** (Java) — Rebuilds the `lite_auto` set using the Java port of Handler D

**Role in pipeline:** Layer 3 in `GlossChainService` — the fastest cache path. The UI is instant when it hits.

### 7. The `contextual_gloss` Table

**Definition (V43):**
```sql
CREATE TABLE contextual_gloss (
    id               BIGSERIAL PRIMARY KEY,
    verse_word_id    BIGINT       NOT NULL REFERENCES verse_word(id) ON DELETE CASCADE,
    language_code    VARCHAR(10)  NOT NULL,
    gloss            VARCHAR(200),
    supplied_gloss   VARCHAR(200),
    is_supplied      BOOLEAN      NOT NULL DEFAULT FALSE,
    model_version    VARCHAR(50)  NOT NULL,
    created_at       TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    UNIQUE (verse_word_id, language_code)
);
```

**What it is:** AI-generated, clause-aware glosses that give each Greek word its most natural English equivalent in the specific verse context, rather than a generic dictionary gloss. Includes `supplied_gloss` (English words added for grammar that have no Greek equivalent) and `is_supplied` flag.

**How it is populated:**
1. **`ContextualGlossService.populateChapter()`** — Generates AI glosses per verse, writes staging file
2. **`ContextualGlossService.commitStaging()`** — After review, upserts rows into the table

**Role in pipeline:** Layer 2.5 in `GlossChainService` — fires ONLY for 13 polysemous lexemes whose meaning depends on clause context.
